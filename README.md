# SDBs — Deemo × アタック25 Discord Bot

リズムゲーム「Deemo」とクイズ番組「アタック25」を掛け合わせた Discord Bot です。
ランダムに選ばれた楽曲のジャケット画像を N 枚 (4/9/16/25) のパネルで覆い、
プレイヤーが Deemo をプレイしてお題をクリアするとパネルが 1 枚ずつめくれ、
徐々に見えてくる楽曲を当てるゲームを進行します。

> 本ドキュメントはゲームを遊ぶプレイヤー向けの説明です。
> セットアップ・運用手順は [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md)、
> 設計詳細は [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) を参照してください。

## ゲームの流れ

1. 誰かが `/start` を実行すると、ランダムな楽曲のジャケットが N 枚のパネルで覆われ、
   N 個のお題と共にチャンネルへ提示されます (このメッセージはピン留めされます)。
2. プレイヤーは実際に Deemo をプレイし、結果を `/play` で入力します。
3. プレイ内容がお題の条件を満たしていれば進捗が `+1` され、お題がクリアされると
   対応するパネルがめくれて楽曲ジャケットが少しずつ見えるようになります。
4. 並行して、メンバーは `/answer` で楽曲名を回答できます。本人だけに正解 / 不正解が
   返されるため、他のプレイヤーには結果が見えません。
5. セッションは **`/end` の実行** または **制限時間 (30 分) の経過** で終了し、
   結果チャンネルへ最終的なパネル画像・スポイラー楽曲名・正解者一覧が投稿されます。


## スラッシュコマンド

| コマンド             | 概要                                                                     |
| :------------------- | :----------------------------------------------------------------------- |
| [`/start`](#start--セッション開始)       | セッションを開始 (楽曲ランダム選択 + お題生成 + パネル画像投稿)   |
| [`/play`](#play--プレイ結果を入力)       | プレイ結果を入力 (お題進捗を自動判定 / パネルめくり)              |
| [`/answer`](#answer--楽曲名を回答)       | 楽曲名を回答 (本人にのみ正解 / 不正解を ephemeral で返答)         |
| [`/progress`](#progress--進捗を確認)     | 現セッションのお題一覧と進捗を表示                                |
| [`/end`](#end--セッション終了)           | セッションを終了 (結果チャンネルへ最終結果を投稿)                 |
| [`/reset`](#reset--セッション強制リセット) | セッションを破棄 (結果チャンネルへの投稿なし。事故時用)         |

同時に存在できるセッションは 1 つだけです。既に進行中のセッションがある状態で `/start` を
実行すると、エラー応答が返ります。

### `/start` — セッション開始

| 引数        | 種別     | 値                                            | 既定        | 説明                                              |
| :---------- | :------- | :-------------------------------------------- | :---------- | :------------------------------------------------ |
| `panels`    | 選択     | `4` / `9` / `16` / `25`                       | `9`         | パネル枚数 (= お題数)                             |
| `rotate`    | 真偽     | `True` / `False`                              | `False`     | 画像を `0` / `90` / `180` / `270` 度のいずれかで回転 |
| `grayscale` | 真偽     | `True` / `False`                              | `False`     | 画像をグレースケール化                            |
| `mosaic`    | 選択     | `なし` / `弱` / `中` / `強` / `最強`          | `なし`      | モザイクの強さ (`最強` が一番ぼやける)            |

- `rotate=True` の場合、`0` 度が選ばれることもあります (見た目は回転なしと同じ)。

### `/play` — プレイ結果を入力

| 引数         | 種別 | 値                                        | 説明                                |
| :----------- | :--- | :---------------------------------------- | :---------------------------------- |
| `song`       | 文字列 | 楽曲名 (オートコンプリート / 部分一致)    | プレイした楽曲                      |
| `difficulty` | 選択 | `Easy` / `Normal` / `Hard` / `Extra`      | プレイした難易度                    |
| `charming`   | 整数 | 1 以上の自然数                            | リザルト画面の charming 数          |
| `combo`      | 整数 | 1 以上の自然数                            | リザルト画面の combo 数             |

以下のいずれかに該当する場合、`/play` は **公開応答せず本人にのみ警告**を返します
(プレイ履歴やお題進捗は更新されません)。

- 楽曲に指定難易度の譜面が存在しない場合
- `charming` がその譜面のノーツ数を超えている場合
- `combo` がその譜面のノーツ数を超えている場合

成功時は進捗のあったお題を embed で公開応答し、必要に応じてピン留め済みメッセージの
パネル画像も差し替えます。

### `/answer` — 楽曲名を回答

| 引数   | 種別   | 説明                                 |
| :----- | :----- | :----------------------------------- |
| `song` | 文字列 | 楽曲名 (オートコンプリート / 部分一致) |

- 結果は本人にのみ ephemeral で返ります。他のメンバーには誰が何を回答したかは見えません。
- 正解しても **セッションは終了しません**。他メンバーが回答するチャンスを保つためです。
- 同じユーザーが正解しても重複登録されず、結果通知の正解者欄に 1 度だけ表示されます。

### `/progress` — 進捗を確認

引数なし。現セッションの全お題と達成数 (`X/Y クリア`) を公開で表示します。
クリア済みのお題は `✅`、未クリアは `⬜` で示されます。

### `/end` — セッション終了

引数なし。進行中セッションを終了し、結果チャンネル (Bot 設置者が設定) へ
[結果通知](#結果通知) を投稿します。同時にピン留めが解除されます。

### `/reset` — セッション強制リセット

引数なし。進行中セッションを **結果通知なしで破棄** し、ピン留めだけを解除します。
誤起動した場合などに使います。

## お題とプレイ品質

各お題には対象とするプレイの「品質」(`play_quality`) が割り当てられています。
品質を満たさないプレイは、そのお題の進捗としてカウントされません。

| 品質   | 条件                                          | 出現比率 |
| :----- | :-------------------------------------------- | :------- |
| `AC`   | charming 数 == その譜面のノーツ数 (All Charming) | 約 10 %  |
| `FC`   | combo 数 == その譜面のノーツ数 (Full Combo)      | 約 30 %  |
| `プレイ` | 任意のプレイ (フィルタなし)                      | 約 60 %  |

お題の表示には品質ラベルが含まれるため、`/start` 直後の embed や `/progress` で
「どの品質のプレイをカウントするか」を確認できます。

お題種別 (タイトル文字一致 / 難易度 / レベル / ノーツ密度 / バージョン / book / shelf …)
の詳細は [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) と
[assets/data/all_topics.json](assets/data/all_topics.json) を参照してください。

## セッションのルール

- **同時 1 セッション**: 既にセッションが進行中なら新規 `/start` は拒否されます。
- **制限時間 30 分**: セッション開始から 30 分経過すると自動終了し、結果通知が投稿されます。
- **残り 10 分前の警告**: 制限時間まで 10 分の時点で、進行中チャンネルに警告 embed が
  送信されます。
- **時間切れ通知**: 制限時間ぴったりに「制限時間が終了しました」という embed が
  進行中チャンネルへ投稿された後、結果チャンネルへ最終結果が送られます。

`session_timeout_minutes` / `warning_minutes_before_end` は Bot 設置者側で
[config/settings.yaml](config/settings.yaml) から変更できます (既定はそれぞれ 30 / 10)。

## Bot からのメッセージ表示

Bot からの応答はすべて **embed 形式** に統一されており、用途別に色分けされています。

| 種別      | 色     | 利用シーン                                                    |
| :-------- | :----- | :------------------------------------------------------------ |
| 情報      | blurple | `/start` でのセッション情報・お題一覧、`/progress` 表示など    |
| 成功      | green   | `/play` で進捗があった応答、`/answer` の正解、`/end` の完了応答 |
| 警告      | orange  | 残り 10 分前の通知、制限時間終了の通知、上限超過の入力拒否     |
| エラー    | red     | バリデーション失敗、`/answer` の不正解、コマンドの例外応答     |

お題一覧の各エントリは `[✅/⬜] パネル {番号} ({進捗}/{必要回数})` という形式の
field 名で並び、本文にはお題の説明文 (品質ラベルや条件が含まれます) が入ります。

## 結果通知

`/end` または 30 分タイムアウトで結果が確定すると、設置者が指定した **結果チャンネル** に
embed で結果が投稿されます。embed には以下が含まれます。

- **タイトル**: `🎵 セッション結果`
- **description**: 終了種別 (`セッション終了` / `セッション終了 (時間切れ)`) と楽曲名
- **楽曲名**: スポイラー記法 (`||...||`) で隠されており、クリックすると展開できます。
  短い楽曲名でも文字数が見抜かれないよう、内側はスペースで 20 文字までパディングされます。
- **画像**: 現在のクリア状況を反映した最終パネル画像 (回転 / グレースケール / モザイクは
  セッション開始時の設定が維持されます)
- **正解者**: `/answer` で楽曲を当てたメンバーの表示名一覧 (アルファベット昇順)。
  0 件のときは `正解者なし` と表示されます。

`/reset` の場合は結果チャンネルへの投稿は行われません。
